库给你自由，框架，给你枷锁。

bootstrap谁用谁知道，bootstrap是为了给缺乏专职前端的团队使用的。所以真正的前端是不愿意用的，不管大项目 还是小项目，再说靠bootstrap搞起来的也不会是什么高端产品，一般就企业应用、内部系统等。

设计师不用 BS，前端怎么用？
设计师用了 BS，还需要前端吗？

BS 只适合两种团队：
没有设计师的团队；
没有前端的团队。

Bootstrap 面向的场景是需要快速搭建整站，或者单独抽取某个模块使用。作为一个 UI 框架，Bootstrap 的组件足够使用，并且也挺美观。

那么为什么会有不愿意用它的情况呢？

因为不适合。如果希望开发长期的项目或者制作产品类网站，那么就需要实现特定的设计，为了在维护项目中可以方便地按设计师要求快速修改样式，肯定会逐步编写出各种业务组件、工具类，相当于为项目自行开发一套框架。这种时候如果继续直接使用 Bootstrap 或其他第三方框架作为底层修改，可能就会面临以下问题：
需要复写大量样式代码，CSS 的层次变得混乱，造成维护上的不便
产生大量冗余代码
项目长期打磨可能引入很多第三方插件，Bootstrap Class-name 命名较为简单，容易引起冲突
因此，是否使用 Bootstrap 完全是取决于项目的实际情况，但在不适合直接使用 Bootstrap 时仍然可以参考 Bootstrap 众多实用的组件，以及 OOCSS 与 SMACSS 概念在其中的实际应用。
